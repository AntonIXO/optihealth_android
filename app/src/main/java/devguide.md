# PIHS: Android Application - Developer Guide

## 1. Project Introduction for the Developer
Welcome to the **Personalized Insight Health Server (PIHS)** project!

**Project Goal:** PIHS aims to empower users by helping them collect, store, and analyze their diverse personal health data to uncover actionable insights. We are building a platform consisting of a Supabase backend, a React-based web frontend for insights and manual entry, and this Android application.

**Role of this Android Application:**
The primary purpose of this Android application is to serve as an automated data synchronization bridge between the user's health data (available via Health Connect by Android) and their personal PIHS account in the Supabase backend.

**Key Responsibilities of the Android App:**
* **User Authentication:** Securely authenticate users against the Supabase backend.
* **Health Data Access:** Integrate with Health Connect by Android to request permissions and read various health metrics (e.g., sleep, steps, heart rate, workouts).
* **Data Transformation:** Map the data retrieved from Health Connect into the standardized PIHS data schema.
* **Secure Data Upload:** Reliably and securely upload the transformed health data to the user's private space within the Supabase (PostgreSQL) database.
* **Background Sync:** Implement mechanisms for periodic and efficient background synchronization of data.

This application is crucial for providing a seamless and automated way for users to populate their PIHS database, which will then be used by the web frontend and analysis engine to generate personalized insights.

Sync State:After a successful upload, record the timestamp of the latest synced data for each data type.
